# 数据加载策略优化方案

## 目标
优化从数据源拉取数据的逻辑，优先拉取本月/本年数据，后台从近到远逐步拉取历年数据。

## 核心策略

### 1. 分层加载优先级

#### 第一优先级：本月数据（实时加载）
- 时间范围：当月 1 日至今
- 加载方式：同步阻塞加载
- 超时时间：5 秒
- 用途：满足日常看盘需求

#### 第二优先级：本年数据（快速加载）
- 时间范围：当年 1 月 1 日至今
- 加载方式：同步阻塞加载
- 超时时间：10 秒
- 用途：满足年度分析需求

#### 第三优先级：历史数据（后台异步加载）
- 时间范围：3 年前、5 年前、全部历史
- 加载方式：后台任务队列异步加载
- 优先级：从近到远（最近 3 年 → 最近 5 年 → 全部）
- 用途：满足回测和深度分析需求

### 2. 数据缓存策略

#### 内存缓存（LRU）
- 本月数据：永久缓存（直到下次启动）
- 本年数据：缓存 24 小时
- 历史数据：缓存 7 天

#### 本地数据库
- 所有数据都持久化到 SQLite
- 按日期分区存储
- 支持增量更新

#### Parquet 冷存储
- 历史数据压缩存储
- 按年度分文件
- 支持快速查询

### 3. API 响应优化

#### 渐进式响应
```json
{
  "status": "partial",  // partial | complete | loading
  "data": [...],        // 已加载的数据
  "coverage": {
    "start_date": "2024-12-01",
    "end_date": "2024-12-10",
    "total_expected": 1000,
    "loaded": 20
  },
  "background_loading": true
}
```

## 技术实现

### 1. 修改数据持久化服务
- 添加日期范围查询优化
- 实现增量数据合并
- 支持后台任务管理

### 2. 后台任务队列
- 使用 asyncio 任务队列
- 支持优先级调度
- 实现断点续传

### 3. 前端适配
- 支持渐进式数据展示
- 显示数据加载进度
- 提供手动刷新选项

## 预期效果

1. **首屏加载时间**：从 30 秒降低到 3 秒
2. **本月数据响应**：< 1 秒
3. **本年数据响应**：< 5 秒
4. **历史数据完整**：后台 5 分钟内完成
