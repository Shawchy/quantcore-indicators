### 1. 统一接口规范

```python
async def get_individual_info(self, code: str) -> Dict[str, Any]:
    """获取个股详细资料"""
    try:
        # 缓存检查
        if self._is_cache_valid(f"individual_info_{code}", ttl=600):
            return self._cache[f"individual_info_{code}"]
        
        # 调用 AkShare API
        data = ak.stock_individual_info_em(symbol=code)
        
        # 数据转换
        result = self._parse_individual_info(data)
        
        # 缓存结果
        self._set_cache(f"individual_info_{code}", result)
        
        return result
        
    except Exception as e:
        logger.error(f"AkShare 获取个股详情失败 {code}: {e}")
        raise
```

from pydantic import BaseModel, Field

class StockIndividualInfo(BaseModel):
    """个股详细资料"""
    code: str = Field(description="股票代码")
    name: str = Field(description="股票简称")
    latest_price: float = Field(description="最新价")
    total_shares: float = Field(description="总股本（亿股）")
    float_shares: float = Field(description="流通股本（亿股）")
    total_market_cap: float = Field(description="总市值（亿元）")
    float_market_cap: float = Field(description="流通市值（亿元）")
    pe_ratio: float = Field(description="市盈率（动态）")
    pb_ratio: float = Field(description="市净率")
    roe: float = Field(description="ROE（加权）")
    eps: float = Field(description="每股收益")
    bps: float = Field(description="每股净资产")
    net_profit: float = Field(description="净利润（亿元）")
    revenue: float = Field(description="营业收入（亿元）")

## ⚠️ 风险评估

### 技术风险

1. **API 稳定性**: AkShare 依赖第三方网站，可能变动
   - **对策**: 多数据源备份、定期测试

2. **反风控**: 频繁请求可能被封 IP
   - **对策**: 频率控制、代理池、缓存优化

3. **数据质量**: 部分数据可能不准确
   - **对策**: 数据验证、异常检测

### 性能风险

1. **请求耗时**: 部分接口响应慢
   - **对策**: 异步请求、缓存、超时控制

2. **内存占用**: 大量数据缓存
   - **对策**: LRU 缓存、定期清理
