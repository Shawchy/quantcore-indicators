## ✅ **替代方案：本地交易日历生成器**

### 实现原理

1. **排除周末**（周六、周日）
2. **排除中国法定节假日**（元旦、春节、清明、劳动节、端午、中秋、国庆）
3. **考虑调休情况**（周末需要上班/交易）

## 📁 **生成的文件**

### 1. CSV 文件

**路径**: `trade_calendar_2024.csv`  
**内容**: 2024 年全年交易日历数据  
**字段**:
- exchange: 交易所代码（SSE）
- cal_date: 日历日期（YYYYMMDD）
- is_open: 是否交易（1=交易，0=休市）
- pretrade_date: 上一个交易日
- reason: 休市原因（节假日/周末）

### 2. JSON 文件

**路径**: `trade_calendar_2024.json`  
**格式**: JSON 数组，包含所有记录

### 3. 后端缓存

**路径**: `backend/data/trading_days_cache.json`  
**用途**: 后端 API 直接读取使用

## 🔧 **集成到后端 API**

### 在 stock_service.py 中使用

```python
from app.services.trade_calendar import get_trading_days

async def get_kline(code: str, start_date: str, end_date: str):
    # 获取交易日
    trading_days = get_trading_days(start_date, end_date)
    
    # 只查询交易日的数据
    klines = await query_kline_in_days(code, trading_days)
    
    return klines
```

### 在 SmartDateSelector.tsx 中使用

```typescript
// 前端加载交易日历缓存
const loadTradingDays = async () => {
  const response = await fetch('/api/v1/trading-days?year=2024');
  const data = await response.json();
  return data.trading_days;
};
```
