# 架构决策记录 (Architecture Decision Records - ADR)

## 📋 什么是 ADR

ADR 是记录系统架构重要决策的文档，包含：
- **背景**: 为什么要做这个决策
- **决策**: 具体选择了什么方案
- **后果**: 这个决策带来的影响（正面和负面）

## 🏗️ 架构总览

```
┌─────────────────────────────────────────────────┐
│              前端层 (React + TypeScript)          │
└──────────────────┬──────────────────────────────┘
                   │ REST API / WebSocket
┌──────────────────▼──────────────────────────────┐
│          API 网关层 (FastAPI)                     │
│  - Swagger 文档                                  │
│  - JWT 认证                                     │

└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│       数据中台调度层 (核心架构)                   │
├─────────────────────────────────────────────────┤
│ 1️⃣ DataSourceFactory - 智能路由                 │
│    - 主备源选择                                 │

├─────────────────────────────────────────────────┤
│ 2️⃣ UnifiedDataAdapter - 统一适配器              │

│    - 质量验证                                   │
│    - 指标计算                                   │
├─────────────────────────────────────────────────┤
│ 3️⃣ StorageRouter - 存储路由                     │
│    - SQLite (热数据 <90 天)                      │
│    - Parquet (温数据 >90 天)                     │
│    - LRU Cache (实时缓存)                        │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│         数据源适配层 (5 个数据源)                  │
│  - EFinance (主力)                               │
│  - AkShare (主力)                                │
│  - Baostock (主力)                               │
│  - TickFlow (可选)                               │
│  - YFinance (禁用)                               │
└─────────────────────────────────────────────────┘
```

## 🎯 核心设计原则

1. **解耦**: 前端、业务、数据源完全解耦
2. **智能**: 自动选择最优数据源，支持故障降级
3. **标准**: 统一数据模型、统一接口规范
4. **分层**: 热/温/冷数据分层存储
5. **扩展**: 易于添加新数据源、新业务

## 📖 ADR 模板

每个 ADR 文档遵循以下结构:

```markdown
# ADR-XXX: 标题

## 状态
[已采纳 | 讨论中 | 已废弃]

## 背景
为什么要做这个决策，问题是什么

## 决策
具体选择了什么方案

## 后果
### 正面影响
- 好处 1
- 好处 2

### 负面影响
- 代价 1
- 代价 2

## 参考链接
- 相关文档
- 代码位置
```
