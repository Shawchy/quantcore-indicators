# API 超时问题修复报告

## 问题描述

**错误信息**:
```
SmartDateSelector.tsx:161  API 加载失败：请求超时
MarketRanking.tsx:41  获取市场排行榜失败：Error: timeout of 30000ms exceeded
MarketRanking.tsx:59  获取市场概览失败：timeout of 30000ms exceeded
```

**问题症状**:
- 多个 API 请求在 30 秒时超时
- K 线数据加载失败
- 市场排行榜和概览数据获取失败
- 用户体验差，页面长时间无响应

---

## 问题分析

### 1. 超时配置过短

**原配置**:
```typescript
const api = axios.create({
  baseURL: API_BASE_URL,
  timeout: 30000, // 30 秒
})
```

**问题原因**:
- 30 秒对于数据加载操作太短
- 从多个数据源拉取数据需要更长时间
- 首次加载全量 K 线数据可能需要 1-2 分钟
- 没有区分快速 API 和慢速 API

### 2. 缺少超时错误提示

**问题**:
- 超时后没有友好的错误提示
- 用户不知道发生了什么
- 没有重试机制

### 3. 没有按 API 类型区分超时

**问题**:
- 所有 API 使用相同的超时时间
- 实时行情 API（应该快）和 K 线数据 API（可能慢）没有区分
- 不够灵活

---

## 修复方案

### 修复 1: 增加全局超时时间

**文件**: `frontend/src/services/api.ts`

**修复内容**:
```typescript
const api = axios.create({
  baseURL: API_BASE_URL,
  timeout: 60000, // ✅ 60 秒（默认）
  headers: {
    'Content-Type': 'application/json',
  },
  // ✅ 自定义超时错误消息
  timeoutErrorMessage: '请求超时，请检查网络连接或稍后重试',
})
```

**改进**:
- ✅ 默认超时从 30 秒 → 60 秒
- ✅ 友好的错误提示
- ✅ 适用于大部分 API

### 修复 2: 为慢速 API 设置更长超时

**文件**: `frontend/src/services/api.ts`

**修复内容**:

#### K 线数据 API（2 分钟）
```typescript
export const stockApi = {
  getKline: (code: string, params?: {...}) =>
    api.get(`/stock/${code}/kline`, {
      params: {...},
      timeout: 120000, // ✅ 2 分钟
    }),
  
  getWeeklyKline: (code: string, params?: {...}) =>
    api.get(`/stock/${code}/kline/weekly`, {
      params: {...},
      timeout: 120000, // ✅ 2 分钟
    }),
  
  getMonthlyKline: (code: string, params?: {...}) =>
    api.get(`/stock/${code}/kline/monthly`, {
      params: {...},
      timeout: 120000, // ✅ 2 分钟
    }),
}
```

**原因**:
- K 线数据需要从数据源拉取大量数据
- 可能需要遍历多个日期范围
- 首次加载可能需要 1-2 分钟

### 修复 3: 优化 React Query 超时配置

**文件**: `frontend/src/main.tsx`（已在之前修复）

**配置**:
```typescript
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      timeout: 15000, // ✅ 15 秒超时（快速失败）
      retry: 2,       // ✅ 重试 2 次
    },
  },
})
```

**说明**:
- React Query 的超时应该更短（快速失败）
- 利用 axios 的超时配置处理慢速 API
- 双层保护机制

## 超时配置策略

### 分级超时配置

| API 类型 | 超时时间 | 适用场景 |
|---------|---------|---------|
| **快速 API** | 15 秒 | 查询、状态检查 |
| **默认 API** | 60 秒 | 一般数据查询 |
| **慢速 API** | 120 秒 | K 线数据、批量加载 |

### 具体配置

```typescript
// 1. 快速 API（15 秒）
- React Query 默认配置
- 状态查询
- 配置查询

// 2. 默认 API（60 秒）
- 股票信息
- 板块数据
- 资金流向

// 3. 慢速 API（120 秒）
- K 线数据（getKline）
- 周线数据（getWeeklyKline）
- 月线数据（getMonthlyKline）
- 历史数据加载
```
