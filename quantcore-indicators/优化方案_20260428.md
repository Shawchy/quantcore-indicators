通过逐行审查 `core.rs` (1802行) 和 `pyo3_bindings.rs` (236行)，发现以下瓶颈：

## 三、性能优化方案

### 3.1 P0: 算法优化（预期 2-10x 加速）

#### 3.1.1 WMA 滑动窗口优化

**现状**: O(n*k) 嵌套循环
```rust
for i in 0..=prices.len() - period {
    let mut weighted_sum = 0.0;
    for j in 0..period {
        weighted_sum += prices[i + j] * (j + 1) as f64;  // 内层循环
    }
    result.push(weighted_sum / weight_sum);
}
```

**优化**: O(n) 滑动窗口
```rust
pub fn wma(prices: &[f64], period: usize) -> Vec<f64> {
    if period < 2 || prices.len() < period { return vec![]; }
    
    let weight_sum = (period * (period + 1) / 2) as f64;
    let mut result = Vec::with_capacity(prices.len() - period + 1);
    
    // 初始窗口
    let mut weighted_sum: f64 = prices[..period]
        .iter().enumerate()
        .map(|(j, &p)| p * (j + 1) as f64).sum();
    result.push(weighted_sum / weight_sum);
    
    // 滑动更新: WMA_new = WMA_old + weight_sum_new - weight_sum_old
    // 推导: WMA[i+1] = WMA[i] + (prices[i+period]*period - sum(prices[i..i+period])) / weight_sum
    let mut window_sum: f64 = prices[..period].iter().sum();
    
    for i in period..prices.len() {
        let outgoing = prices[i - period];
        let incoming = prices[i];
        window_sum = window_sum - outgoing + incoming;
        weighted_sum = weighted_sum - window_sum + incoming * period as f64;
        result.push(weighted_sum / weight_sum);
    }
    
    result
}
```

**预期**: 10万条×20周期: 从 2.73ms → ~0.3ms (**9x 加速**)

#### 3.1.2 EMA 缓存友好优化

**现状**: `result.last().unwrap()` 每次间接访问
```rust
let prev_ema = *result.last().unwrap();
```

**优化**: 使用局部变量
```rust
pub fn ema(prices: &[f64], period: usize) -> Vec<f64> {
    let multiplier = 2.0 / (period as f64 + 1.0);
    let mut result = Vec::with_capacity(prices.len() - period + 1);
    
    let mut current_ema: f64 = prices[..period].iter().sum::<f64>() / period as f64;
    result.push(current_ema);
    
    for i in period..prices.len() {
        current_ema = (prices[i] - current_ema) * multiplier + current_ema;
        result.push(current_ema);
    }
    
    result
}
```

**预期**: 消除 Vec 间接访问，约 **1.5-2x 加速**

#### 3.1.3 CCI 增量绝对偏差优化

**现状**: O(n*k) 内层循环计算 mean_deviation
```rust
let mean_deviation: f64 = tp[i..i + period]
    .iter().map(|&x| (x - avg_tp).abs()).sum::<f64>() / period as f64;
```

**优化**: 使用滑动窗口维护 sum_abs_dev
```rust
// 初始化
let mut sum_tp: f64 = tp[..period].iter().sum();
let avg_tp = sum_tp / period as f64;
let mut sum_abs_dev: f64 = tp[..period].iter().map(|&x| (x - avg_tp).abs()).sum();

for i in 0..=tp.len() - period {
    let avg_tp = sum_tp / period as f64;
    let mean_deviation = sum_abs_dev / period as f64;
    
    // ... 计算 CCI ...
    
    // 滑动更新 sum_abs_dev
    if i + period < tp.len() {
        let old_avg = sum_tp / period as f64;
        sum_tp = sum_tp - tp[i] + tp[i + period];
        let new_avg = sum_tp / period as f64;
        
        // 近似更新: 移除旧元素的绝对偏差，添加新元素的绝对偏差
        sum_abs_dev = sum_abs_dev 
            - (tp[i] - old_avg).abs() 
            + (tp[i + period] - new_avg).abs();
    }
}
```

**预期**: 从 O(n*k) → O(n)，约 **5-10x 加速**

#### 3.1.4 Bollinger 消除重复计算

**现状**: 调用 `ma()` 后又独立计算 `sum`
```rust
let middle = ma(prices, period);  // ma() 内部已计算 sum
let mut sum: f64 = prices[..period].iter().sum();  // 重复计算
```

**优化**: 合并为单次遍历
```rust
pub fn bollinger_bands(prices: &[f64], period: usize, std_dev: f64) -> BollingerBandsResult {
    let mut sum = prices[..period].iter().sum::<f64>();
    let mut sum_sq = prices[..period].iter().map(|&x| x * x).sum::<f64>();
    
    let mut upper = Vec::with_capacity(prices.len() - period + 1);
    let mut middle = Vec::with_capacity(prices.len() - period + 1);
    let mut lower = Vec::with_capacity(prices.len() - period + 1);
    
    for i in 0..=prices.len() - period {
        let mean = sum / period as f64;
        let variance = (sum_sq / period as f64 - mean * mean).max(0.0);
        let std = variance.sqrt();
        
        middle.push(mean);
        upper.push(mean + std_dev * std);
        lower.push(mean - std_dev * std);
        
        if i + period < prices.len() {
            sum = sum - prices[i] + prices[i + period];
            sum_sq = sum_sq - prices[i] * prices[i] + prices[i + period] * prices[i + period];
        }
    }
    
    BollingerBandsResult { upper, middle, lower }
}
```

**预期**: 消除 ma() 调用开销，约 **1.5x 加速**

### 3.2 P1: SIMD 向量化（预期 2-4x 加速）

使用 Rust `std::simd` (nightly) 或 `wide` crate (stable):

```rust
use wide::f64x4;

pub fn ma_simd(prices: &[f64], period: usize) -> Vec<f64> {
    if period < 2 || prices.len() < period { return vec![]; }
    
    let mut result = Vec::with_capacity(prices.len() - period + 1);
    let inv_period = 1.0 / period as f64;
    
    let mut sum: f64 = prices[..period].iter().sum();
    result.push(sum * inv_period);
    
    // SIMD 批量处理: 一次处理 4 个滑动窗口
    let mut i = period;
    while i + 4 <= prices.len() {
        let s0 = sum - prices[i - period] + prices[i];
        let s1 = s0 - prices[i - period + 1] + prices[i + 1];
        let s2 = s1 - prices[i - period + 2] + prices[i + 2];
        let s3 = s2 - prices[i - period + 3] + prices[i + 3];
        
        let sums = f64x4::new([s0, s1, s2, s3]);
        let inv = f64x4::splat(inv_period);
        let results = sums * inv;
        
        result.extend_from_slice(&results.to_array());
        sum = s3;
        i += 4;
    }
    
    // 处理剩余元素
    for j in i..prices.len() {
        sum = sum - prices[j - period] + prices[j];
        result.push(sum * inv_period);
    }
    
    result
}
```

**预期**: MA/EMA 等简单指标 **2-4x 加速**

### 3.4 P3: 流式/增量计算（实时场景关键优化）

```rust
pub struct EmaState {
    value: f64,
    multiplier: f64,
    initialized: bool,
}

impl EmaState {
    pub fn new(period: usize) -> Self {
        Self { value: 0.0, multiplier: 2.0 / (period as f64 + 1.0), initialized: false }
    }
    
    pub fn update(&mut self, price: f64) -> f64 {
        if !self.initialized {
            self.value = price;
            self.initialized = true;
        } else {
            self.value = (price - self.value) * self.multiplier + self.value;
        }
        self.value
    }
}
```

**预期**: 实时行情场景，每次新数据仅需 O(1) 计算

## 八、总结

### 核心问题

1. **WMA/CCI 算法未优化**: O(n*k) 嵌套循环，是最大性能瓶颈
2. **简单指标不如 pandas**: EMA/VWAP/MACD 缓存不友好
3. **指标数量不足**: 14 个 vs pandas-ta 121+，覆盖率 11.6%

### 优化策略

1. **算法优先**: WMA 滑动窗口 + CCI 增量偏差 → 最大收益
2. **SIMD 第二**: MA/EMA 向量化 → 简单指标超越 pandas
3. **并行第三**: rayon 多指标并行 → 整体吞吐量提升
4. **流式第四**: 增量计算 → 实时场景关键
