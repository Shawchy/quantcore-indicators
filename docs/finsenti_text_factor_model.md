### 阶段 2: 因子工程化（2-3 周）

```python
# 因子标准化流程
class TextFactorPipeline:
    def __init__(self):
        self.finsenti = FinSentiModel()
    
    def process(self, texts: List[str]) -> pd.DataFrame:
        # 1. 情感分析
        factors = self.finsenti.analyze_batch(texts)
        
        # 2. Z-Score 标准化
        factors_z = (factors - factors.mean()) / factors.std()
        
        # 3. 行业中性化
        factors_neutral = self.industry_neutral(factors_z)
        
        # 4. 市值中性化
        factors_final = self.market_cap_neutral(factors_neutral)
        
        return factors_final
```

### 阶段 3: 多因子整合（3-4 周）

```python
# 整合到现有量化系统
class MultiFactorModel:
    def __init__(self):
        # 传统因子
        self.value_factor = ValueFactor()
        self.momentum_factor = MomentumFactor()
        self.quality_factor = QualityFactor()
        
        # 文本因子 (新增)
        self.text_factor = TextFactorPipeline()
    
    def predict(self, stocks: List[str]) -> pd.Series:
        # 传统因子
        signals = (
            0.3 * self.value_factor.calculate(stocks) +
            0.3 * self.momentum_factor.calculate(stocks) +
            0.2 * self.quality_factor.calculate(stocks) +
            0.2 * self.text_factor.calculate(stocks)  # 新增 20% 权重
        )
        
        return signals
```

## ✅ 为什么这样设计？

### 1. **专业的人做专业的事**
```
FinSenti: 文本 → 因子 (专精)
Qwen3.5:  因子 → 决策 (综合)
```

### 2. **避免互相干扰**
```
❌ 错误做法：让主模型同时处理技术和文本
   → 注意力分散，两者都不精

✅ 正确做法：专用模型处理专用任务
   → 各展所长，效果最优
```

### 3. **因子可解释性**
```
文本因子独立输出:
- 可回溯测试
- 可归因分析
- 可调整权重
- 可单独优化
```

### 4. **系统可扩展性**
```
未来可增加:
- 财报因子专用模型
- 宏观因子专用模型
- 行业轮动专用模型

主模型只需整合因子，无需重新训练
```
