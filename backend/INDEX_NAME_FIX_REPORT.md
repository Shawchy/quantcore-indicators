# 指数名称修复报告

## 问题描述

用户反馈：**"目前大盘时时行情中指数名称为 指数 000001 指数 399001 指数 修改成正确的上证指数 深证成指 创业板指"**

问题现象：
- ❌ 上证指数显示为"指数 000001"
- ❌ 深证成指显示为"指数 399001"
- ❌ 创业板指显示为"指数 399006"
- ❌ 其他指数也使用代码而非中文名称

## 修复方案

### 修改文件

`app/adapters/efinance_adapter.py` (第 1410-1445 行)

### 修改内容

在指数处理逻辑中添加名称映射表 `name_map`，将指数代码映射到中文名称：

```python
name_map = {
    '000001': '上证指数',
    '399001': '深证成指',
    '399006': '创业板指',
    '000016': '上证 50',
    '000300': '沪深 300',
}
```

修改返回数据中的名称字段：

```python
# 修改前
'name': f"指数{code}"

# 修改后
'name': name_map.get(code, f"指数{code}")
```

## 技术实现

### 1. 名称映射表

在 `symbol_map` 的基础上增加了 `name_map`：

```python
symbol_map = {
    '000001': 'sh000001',  # 用于 akshare 数据获取
    '399001': 'sz399001',
    '399006': 'sz399006',
    '000016': 'sh000016',
    '000300': 'sh000300',
}

name_map = {
    '000001': '上证指数',  # 用于前端显示
    '399001': '深证成指',
    '399006': '创业板指',
    '000016': '上证 50',
    '000300': '沪深 300',
}
```

### 2. 容错处理

使用 `name_map.get(code, f"指数{code}")` 确保：
- 如果在映射表中找到代码，返回中文名称
- 如果不在映射表中，返回默认格式"指数 XXXXXX"

### 3. 缓存清理

运行 `clear_index_cache.py` 清除所有指数缓存，确保立即生效。
