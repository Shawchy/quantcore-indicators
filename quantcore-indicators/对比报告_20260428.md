### 性能分级

**🚀🚀 显著领先 (50x+)**：WMA (64x)、CCI (73x)
- 原因：pandas 使用 `rolling.apply(lambda)` 每次调用 Python 函数
- Rust 直接循环，避免 Python 解释器开销

**⬆️ 明显领先 (1.5-3x)**：ADX、ATR、OBV、RSI、Bollinger
- 原因：复杂指标需多步操作，Rust 单遍完成
- pandas 需要多次 rolling/ewm/where 操作

**≈ 性能相当 (0.6-1.1x)**：MA、EMA、MACD、KDJ、Williams %R、Stochastic、VWAP
- 原因：pandas 底层使用 C 级优化的 rolling/ewm
- Rust 优势被 numpy C 实现抵消

## 三、Rust 代码性能分析

### 已优化项

| 优化 | 状态 | 效果 |
|------|------|------|
| 单调队列滑动窗口 (KDJ/Stochastic) | ✅ O(n) | vs pandas O(n*k) |
| 预计算典型价格 (CCI) | ✅ | 避免重复计算 |
| 滑动窗口求和 (MA/Bollinger) | ✅ O(n) | 避免重复求和 |
| LTO + codegen-units=1 | ✅ | 极致编译优化 |
| numpy 零拷贝 | ✅ | 避免 list↔Vec 转换 |

### 剩余优化空间

| 优化 | 预期效果 | 工作量 |
|------|---------|--------|
| SIMD 向量化 (MA/EMA/MACD) | 2-4x 加速 | 中等 |
| 多线程计算 (大数据量) | 4-8x 加速 (多核) | 较大 |
| 内存池预分配 | 10-20% 提升 | 小 |
| 数值稳定性 (Kahan 求和) | 精度提升 | 小 |
| Arrow 零拷贝后端 | 30-50% 数据传输优化 | 中等 |

### 性能瓶颈根因

1. **单线程计算**: Rust 代码是单线程，无法利用多核 CPU
2. **numpy 中间分配**: `into_pyarray()` 需要分配新 numpy 数组
3. **算法差异**: 简单指标 pandas C 级 rolling 已非常优化

```python

fallback → pandas-ta                            # 稀有指标
```

## 六、总结

### Rust 核心优势

1. **极致性能**: WMA (64x)、CCI (73x) 显著领先
2. **算法优化**: 单调队列 O(n) vs pandas O(n*k)
3. **工程可靠**: 59+ 测试用例，编译期安全
4. **无缝集成**: PyO3 + numpy 零拷贝
5. **fallback 机制**: Rust 失败自动降级 pandas

### pandas-ta 核心优势

1. **指标丰富**: 121+ 指标全市场覆盖
2. **生态成熟**: 社区活跃，持续更新
3. **C 级优化**: rolling/ewm 底层 C 实现
4. **无需编译**: pip install 即用

### 结论

**quantcore-indicators 在核心指标上具有显著性能优势**（WMA/CCI 60-70x 加速），但在指标种类上仅覆盖 11.6%。

**最佳实践**: 两者互补使用
- 核心高频指标 (MA/EMA/RSI/KDJ/WMA/CCI) → Rust
- 稀有指标 (Ichimoku/Fibonacci/Candle) → pandas-ta

通过 `indicators_manager.py` 的智能选择机制，既保证了核心指标的高性能，又保留了 pandas-ta 的指标丰富度。
