# ADR-003: 统一数据模型设计

## 状态
✅ **已采纳** (2026-03-28)

## 背景

### 问题陈述
在接入多个数据源时，面临数据格式不统一的问题：

1. **字段命名不一致**: 
   - EFinance: `open`, `high`, `low`, `close`
   - AkShare: `开盘价`, `最高价`, `最低价`, `收盘价`
   - Baostock: `open`, `high`, `low`, `close`, `preclose`

2. **数据格式差异**:
   - 日期格式：`2024-01-01` vs `20240101` vs `2024/01/01`
   - 股票代码：`000001` vs `000001.SZ` vs `sz000001`
   - 数值精度：浮点数精度不一致

3. **数据质量参差**:
   - 部分数据源返回空值
   - 价格异常（高 < 低）
   - 成交量为负数

4. **业务逻辑耦合**:
   - 业务代码需要处理多种数据格式
   - 新增数据源需要修改大量业务代码

### 需求分析
- 统一数据格式和字段命名
- 数据质量验证和清洗
- 业务逻辑与数据源解耦
- 支持数据质量评分

## 决策

### 选择方案
**设计统一的 Pydantic 数据模型**，作为所有数据源的标准输出格式。

### 核心模型设计

#### 1. UnifiedKLine - 统一 K 线模型
```python
# backend/app/models/unified_models.py
class UnifiedKLine(BaseModel):
    """统一 K 线数据模型"""
    # 基础信息
    code: str = Field(..., description="股票代码（6 位数字）")
    date: str = Field(..., description="日期（YYYY-MM-DD）")
    time: Optional[str] = Field(None, description="时间")
    
    # 行情数据
    open: float = Field(..., ge=0, description="开盘价")
    high: float = Field(..., ge=0, description="最高价")
    low: float = Field(..., ge=0, description="最低价")
    close: float = Field(..., ge=0, description="收盘价")
    pre_close: Optional[float] = Field(None, description="昨收价")
    
    # 成交量和成交额
    volume: float = Field(..., ge=0, description="成交量（股）")
    amount: Optional[float] = Field(None, description="成交额（元）")
    turnover_rate: Optional[float] = Field(None, description="换手率（%）")
    
    # 复权信息
    adjust_type: AdjustType = Field(AdjustType.QFQ, description="复权类型")
    adjust_factor: Optional[float] = Field(1.0, description="复权因子")
    
    # 数据来源
    source: DataSourceType = Field(..., description="数据源")
    source_time: Optional[str] = Field(None, description="数据源时间戳")
    
    # 质量评分
    quality_score: float = Field(1.0, ge=0, le=1, description="数据质量评分")
    
    # 验证器
    @field_validator('code')
    def validate_code(cls, v):
        if not v.isdigit() or len(v) != 6:
            raise ValueError("股票代码必须是 6 位数字")
        return v
    
    @field_validator('date')
    def validate_date(cls, v):
        try:
            datetime.strptime(v, "%Y-%m-%d")
            return v
        except ValueError:
            raise ValueError("日期格式必须是 YYYY-MM-DD")
```

#### 2. 数据标准化器
```python
# backend/app/utils/data_normalizer.py
class DataNormalizer:
    @staticmethod
    def normalize_kline(data: Dict, source_type: DataSourceType) -> Optional[UnifiedKLine]:
        """将原始数据标准化为 UnifiedKLine"""
        
        # 字段映射转换
        field_mapping = {
            DataSourceType.AKSHARE: {
                '开盘价': 'open',
                '最高价': 'high',
                '最低价': 'low',
                '收盘价': 'close',
            },
            DataSourceType.BAOSTOCK: {
                'open': 'open',
                'high': 'high',
                # ...
            }
        }
        
        # 日期格式统一
        date = DataNormalizer.normalize_date(data.get('date'))
        
        # 股票代码统一
        code = DataNormalizer.normalize_code(data.get('code'))
        
        # 创建统一模型
        kline = UnifiedKLine(
            code=code,
            date=date,
            open=float(data['open']),
            high=float(data['high']),
            low=float(data['low']),
            close=float(data['close']),
            volume=float(data['volume']),
            source=source_type,
            quality_score=1.0
        )
        
        return kline
```

#### 3. 数据验证器
```python
# backend/app/utils/data_normalizer.py
class DataNormalizer:
    @staticmethod
    def validate_kline(kline: UnifiedKLine) -> bool:
        """验证 K 线数据质量"""
        
        # 价格合理性检查
        if not (kline.high >= kline.low):
            logger.warning(f"价格异常：high < low")
            return False
        
        if not (kline.high >= kline.open and kline.high >= kline.close):
            logger.warning(f"价格异常：high 不是最高")
            return False
        
        if not (kline.low <= kline.open and kline.low <= kline.close):
            logger.warning(f"价格异常：low 不是最低")
            return False
        
        # 成交量检查
        if kline.volume < 0:
            logger.warning(f"成交量为负数")
            return False
        
        # 涨跌幅检查（A 股涨跌停限制）
        if kline.pre_close:
            change_pct = abs(kline.close - kline.pre_close) / kline.pre_close
            if change_pct > 0.11:  # 超过 11% 视为异常
                logger.warning(f"涨跌幅异常：{change_pct*100:.2f}%")
                return False
        
        return True
```

## 后果

### 正面影响

1. **业务解耦**
   - ✅ 业务代码无需关心数据源差异
   - ✅ 新增数据源不影响现有业务
   - ✅ 代码可维护性大幅提升

2. **数据质量提升**
   - ✅ 统一验证标准
   - ✅ 异常数据自动识别
   - ✅ 质量评分可追溯

3. **开发效率**
   - ✅ 统一接口规范
   - ✅ 减少重复代码
   - ✅ IDE 自动补全支持

4. **类型安全**
   - ✅ Pydantic 类型检查
   - ✅ 运行时数据验证
   - ✅ 减少类型错误

5. **文档化**
   - ✅ 模型即文档
   - ✅ 自动生成 API 文档
   - ✅ 字段含义清晰

### 负面影响

1. **开发成本**
   - ❌ 需要定义 100+ 模型
   - ❌ 需要实现字段映射
   - ❌ 需要编写验证逻辑

2. **性能开销**
   - ❌ 数据转换需要时间
   - ❌ 验证逻辑增加延迟
   - ⚠️ 影响较小 (<5ms)

3. **维护成本**
   - ❌ 模型变更需要同步更新
   - ❌ 字段映射需要维护
   - ❌ 验证规则需要优化
