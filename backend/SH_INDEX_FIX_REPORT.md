## 📊 问题诊断

### 问题现象
前端显示上证指数为 **11.02 点**，而实际应该在 **3900 点** 左右。

### 根本原因
1. **数据源问题**: efinance 数据源将代码 `000001` 识别为"平安银行"股票，而不是上证指数
2. **缓存问题**: 数据库中保存了错误的实时行情数据（平安银行的股票数据）
3. **指数支持缺失**: efinance 适配器不支持指数实时行情获取

### 问题链路
```
前端请求上证指数 (000001)
  ↓
efinance 数据源返回平安银行数据 (11.02 元)
  ↓
保存到数据库 (price=11.02)
  ↓
后续请求直接返回缓存的错误数据
```

---

## ✅ 解决方案

### 1. 修改 efinance 适配器
修改了 [`app/adapters/efinance_adapter.py`](file:///m:/Project/Quant/backend/app/adapters/efinance_adapter.py) 的 `get_realtime_quote` 函数：

**添加指数检测逻辑**:
```python
# 检测是否为指数代码（000001=上证指数，399001=深证成指等）
# efinance 不支持指数实时行情，使用 akshare 替代
if code in ['000001', '399001', '399006', '000016', '000300']:
    import akshare as ak
    # 使用 akshare 获取指数实时行情
    symbol_map = {
        '000001': 'sh000001',  # 上证指数
        '399001': 'sz399001',  # 深证成指
        '399006': 'sz399006',  # 创业板指
        '000016': 'sh00016',   # 上证 50
        '000300': 'sh000300',  # 沪深 300
    }
    ak_symbol = symbol_map.get(code, f'sh{code}')
    df = ak.stock_zh_index_daily(symbol=ak_symbol)
    # ... 返回指数数据
```

### 2. 清除错误缓存数据
删除了数据库中 `000001` 的错误记录：
```sql
DELETE FROM realtime_quote WHERE code = '000001';
```

### 3. 重启后端服务
重启服务以应用代码修改。

## 🛠️ 技术细节

### 数据源切换逻辑
```
请求指数行情 (000001)
  ↓
efinance 适配器检测到是指数代码
  ↓
自动切换到 akshare 数据源
  ↓
调用 ak.stock_zh_index_daily('sh000001')
  ↓
返回最新收盘价作为当前价
  ↓
保存到数据库并返回给前端
```

### akshare API
使用 `ak.stock_zh_index_daily` 获取指数历史行情，取最新一条数据：
```python
df = ak.stock_zh_index_daily(symbol='sh000001')
row = df.iloc[-1]  # 取最新一行
price = float(row['close'])  # 收盘价
```
